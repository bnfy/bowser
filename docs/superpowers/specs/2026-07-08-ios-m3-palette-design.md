# iOS M3: Island Palette & Quick Switcher

Milestone 3 of the iOS port roadmap. Adds the island's expanded state: tap
the pill to open a half-sheet palette with a combined address bar, tab
switcher, slash-command filtering, and Quick Switcher. Builds on the M0–M2
walking skeleton (TabsManager, TabModel, ContentView with address pill + tab
dots, TabListSheet).

Maps to **F1** (island expanded state), **F6** (Quick Switcher, basic — open
tabs only), **F7** (slash commands, subset), **D7** (hardware-keyboard
shortcuts deferred to post-parity). Desktop reference: `src/renderer/overlay.js`
(command bar, panel/palette modes, Quick Switcher matching), `src/main/main.js`
(`showOverlay`/`hideOverlay`, `overlayMode`).

## Pill becomes display-only

The pill changes from an interactive address bar to a compact status display.
All text input moves to the palette sheet.

**Removed from the pill:**
- The `TextField` — replaced by a `Text` showing the current page's domain
  (or "New Tab" for blank tabs), truncated naturally.
- Back/forward buttons — iOS users navigate with edge swipes via
  `WKWebView.allowsBackForwardNavigationGestures`. No buttons needed.

**Retained on the pill (direct-action, no sheet needed):**
- Tab dots (visual indicators, capped at 3, `+N` overflow) — unchanged from M2.
- Reload/stop button.
- `+` new-tab button.

**New behavior:** tapping anywhere in the pill's middle zone (dots + domain
text) opens the palette sheet. The reload/stop and `+` buttons are excluded
from this tap target — they act directly.

**Liquid Glass:** on iOS 26+, the pill uses `.glassEffect(.regular.interactive,
in: .capsule)` instead of the opaque token surface. On earlier iOS versions
(the deployment target is iOS 17), the pill falls back to the existing
`BlancTokens.surfaceRaised` background + `BlancTokens.border` stroke — an
`#available(iOS 26, *)` compile-time gate, not a runtime feature flag. This
adapts to light/dark/private automatically and feels native on the latest OS
while remaining functional on the floor. Deliberate iOS divergence from the
desktop's opaque token-defined surface — recorded as **D15** in the divergence
register (added alongside this spec).

The resting pill layout:

```
[ ● ●● ] [ example.com        ] [ ↻ ] [ + ]
```

## Palette sheet

Presented as a SwiftUI `.sheet` with `.presentationDetents([.medium, .large])`,
defaulting to `.medium`. The sheet replaces `TabListSheet` — the standalone
tab-list sheet from M2 is deleted.

### Input field

A `TextField` at the top of the sheet, auto-focused when the sheet appears.
It serves triple duty:

1. **Address/search input** — tapping Go resolves per **Submit behavior**
   below (may navigate, switch tabs, execute a command, or no-op).
2. **Slash-command prefix** — when the input starts with `/`, the list below
   switches to filtered slash commands.
3. **Quick Switcher query** — any other text triggers the Quick Switcher,
   searching open tabs by title and URL.

### Submit behavior (Go / Enter)

Ported from the desktop's `addressInput` keydown handler in `overlay.js`.
The input's submit action resolves in priority order:

1. **Slash mode with matches** — executes the top matching command, dismisses.
2. **Switcher mode with a strong match** (score >= `STRONG_MATCH_SCORE`, i.e.
   a substring hit) — picks the top result (switches to that tab), dismisses.
   Weak in-order matches (score 1) stay visible and tappable but are never
   auto-selected — they're too permissive.
3. **Slash mode with no matches** — does nothing. A typo like `/tyop` doesn't
   get treated as an address or search.
4. **Anything else** (no switcher match, or the input is plain text) — calls
   `submitAddress(using:)` on the active tab (same normalization as M1),
   dismisses.

### Three list modes

The list below the input switches reactively based on the input text:

**1. Resting state (input empty)** — shows the open-tab list. Same content
as M2's `TabListSheet`: each row shows the tab's title (or "New Tab"), URL,
and a checkmark on the active tab. Swipe-to-delete closes a tab. Tapping a
row switches to that tab and dismisses.

**2. Slash-command mode (input starts with `/`)** — filters and shows
matching commands. Each row shows the command name and a one-line hint
(sourced from `copy/slash-commands.json`). Tapping a row executes the command
and dismisses. At M3, only two commands are executable:

| Command | Hint (from `SlashCommands.strings`) | Action |
|---------|--------------------------------------|--------|
| `/new` | `slash_new`: "Open a new tab" | `TabsManager.createTab()` |
| `/close` | `slash_close`: "Close this tab" | `TabsManager.closeTab(activeTabId)` |

Hints are keyed from `copy/generated/SlashCommands.strings` (the iOS
artifact generated by `copy/build.mjs` from `copy/slash-commands.json`) —
not hand-written in the Swift source. `SlashCommand`'s static list references
these string keys so copy stays in sync with the substrate.

Commands whose backing features don't exist yet (e.g. `/favorites`,
`/private`, `/find`) are **not shown**. The command list grows as milestones
land features — each milestone adds its commands to the `available` list. No
greyed-out or "coming soon" states.

**3. Quick Switcher mode (any other text)** — searches open tabs using the
desktop's two-tier matching algorithm. Results are sorted by score descending,
then by position in the tab list. Tapping a result switches to that tab and
dismisses.

### Dismiss behavior

The sheet dismisses on:
- Tapping a tab row (switches to it).
- Tapping a slash-command row (executes it).
- Tapping a Quick Switcher result (switches to it).
- Submitting the input (navigates the active tab).
- Swiping down (standard sheet dismiss).

## Quick Switcher matching

Ported from `overlay.js` `matchScore()` + `switcherResults()`. Two scoring
tiers:

1. **Substring match (score 2)** — the query appears as a contiguous
   case-insensitive substring in the candidate's matchable text.
2. **In-order character match (score 1)** — each character of the query
   appears in order (not necessarily contiguous) in the matchable text.
   E.g. "gml" matches "gmail.com".

**Matchable text** follows the desktop's `matchableText()`: title + host +
capped path (first 64 chars). Query strings and fragments are excluded —
OAuth/token URLs carry kilobytes of base64 that in-order-match almost any
query (this was a real desktop bug).

**Kind bonuses** (desktop pattern, for future extensibility):
- Tabs: `score + 0.2`

At M3 only tabs are searched, so the bonus is inert — but the scoring
structure is laid in so that adding favorites (+0.1), history (+0.0), and
groups (+0.3) in later milestones doesn't require restructuring.

**Dedup:** results are deduped by tab id (same as desktop's `seen` Set).

**Cap:** max 6 results (matches desktop).

**STRONG_MATCH_SCORE = 2:** only substring-tier matches are confident enough
to auto-navigate on bare Enter. The in-order fallback is too permissive for
that.

## Architecture

### New files

**`PaletteSheet.swift`** — the half-sheet view. Holds the input `TextField`,
determines the current mode (resting/slash/switcher) from the input text, and
renders the appropriate list section. Receives `TabsManager` so it can read
tabs, execute slash commands, and call `submitAddress`. Dismisses itself after
every action.

**`QuickSwitcher.swift`** — a pure struct with a single static method:
`search(query:tabs:) -> [SwitcherResult]`. Contains `matchScore()`,
`matchableText()`, scoring, dedup, sort, and cap logic. No UI, no state —
extracted for testability. `SwitcherResult` carries `tab: TabModel`,
`score: Double`, and a `title`/`subtitle` for display.

**`SlashCommand.swift`** — defines the `SlashCommand` struct (name, hint key)
and a static `available` list. At M3 this is two entries. Hints are loaded
from `SlashCommands.strings` (the generated iOS artifact from
`copy/slash-commands.json`) via `NSLocalizedString` keyed by
`slash_new`/`slash_close`. Each command carries an `execute(manager:)`
closure. Filtering is a simple `command.hasPrefix(query)` match.

### Modified files

**`ContentView.swift`** — pill becomes display-only: `TextField` replaced by
a tappable `Text` domain display, back/forward buttons removed,
`.glassEffect()` applied. `showTabList` state renamed to `showPalette`.
The `.sheet` now presents `PaletteSheet` instead of `TabListSheet`.

**`TabModel.swift`** — `webView.allowsBackForwardNavigationGestures = true`
set at init (replaces the removed back/forward buttons).

**`TabListSheet.swift`** — deleted. Its content (tab rows, swipe-to-delete)
moves into `PaletteSheet`'s resting-state list.

### Data flow

```
Tap pill middle zone
  → showPalette = true
  → PaletteSheet presented (.medium detent)
  → Input auto-focused
  → User types:
      "" (empty)   → tab list (resting)
      "/" prefix    → slash-command filter
      anything else → QuickSwitcher.search()
  → User acts:
      tap tab row       → manager.setActive(id), dismiss
      tap command row   → command.execute(manager:), dismiss
      tap switcher row  → manager.setActive(id), dismiss
      submit input      → resolve per Submit behavior matrix
      swipe-delete row  → manager.closeTab(id)
      swipe down        → dismiss (no action)
```

## What is NOT in M3

Deferred to later milestones per the roadmap:

- Favorites/history in Quick Switcher (M7)
- Group names in Quick Switcher (M8)
- Slash commands beyond `/new` and `/close` (added per-milestone)
- Private tabs (M9 — `/private` command added then)
- Full slash-command set + Find in page (M10 — `/find` and remaining
  commands added then)
- `blanc://` newtab page (M4)
- Hardware-keyboard shortcuts for opening the palette (D7, post-parity)

## Tests

Unit tests in `BlancTests/`:

- **`QuickSwitcherTests`** — substring match returns score 2, in-order match
  returns score 1, no-match excluded, case insensitivity, matchable-text
  strips query strings, results sorted by score, cap at 6, dedup by tab id.
- **`SlashCommandTests`** — filtering by prefix, `/new` and `/close` present,
  empty prefix returns all, non-matching prefix returns empty.

No UI tests (consistent with M0–M2 and the desktop project). Palette is
verified visually on the simulator.
