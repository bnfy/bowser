# Arrow-key workspace navigation + Help menu with Keyboard Shortcuts page

**Date:** 2026-07-06
**Status:** Approved

Two related features:

1. **Arrow-key navigation** — ⌥⌘←/→ flips between tabs within the current
   cluster; ⌥⌘↑/↓ jumps between clusters (groups).
2. **Help menu** — a new native Help menu whose "Keyboard Shortcuts" item
   opens a new `blanc://shortcuts/` internal page listing every keystroke,
   generated by introspecting the live application menu so it can't drift.

## Part 1: Arrow-key workspace navigation

### Shortcuts

All four are global, registered as visible menu accelerators (the same
mechanism as every existing shortcut), so they work regardless of which view
has focus and are discoverable in the menu bar. Bare arrows are untouched —
web content keeps them.

| Keys | Action |
|---|---|
| ⌥⌘← / ⌥⌘→ (`Alt+CmdOrCtrl+Left/Right`) | Previous/next tab within the active tab's cluster, wrapping |
| ⌥⌘↑ / ⌥⌘↓ (`Alt+CmdOrCtrl+Up/Down`) | Previous/next cluster, wrapping; lands on the cluster's last-active tab |

⌥⌘←/→ is Safari's and Chrome's macOS tab-switch combo, so it feels native;
Blanc scopes it to the cluster.

### Definitions

A **cluster** is the same unit `selectTabAtIndex` (⌘1–9) already uses, in the
same order: a leading pseudo-cluster of pinned tabs (if any), then each
non-empty group in `groups` order (via `clusterList()`), then a trailing
pseudo-cluster of ungrouped tabs. Cluster keys for state: the group `id`,
`'pinned'`, or `'loose'`.

### Behavior

- **Within-cluster (←/→):** cycle through the active cluster's `tabIds` with
  wrap-around, same convention as `cycleTab`. Single-tab cluster → no-op.
  **No groups and no pinned tabs → delegate to the existing `cycleTab()`**
  (degrades to plain tab cycling, same as Ctrl+Tab).
- **Between clusters (↑/↓):** step ±1 through the slots list with wrap.
  Only one cluster → no-op. Landing on a cluster activates its
  **last-active tab**, falling back to its first tab; landing on a collapsed
  group uncollapses it (consistent with `focusGroup`).
- **Last-active memory:** a module-level `Map<clusterKey, tabId>` in
  `main.js`, updated inside `setActiveTab`. In-memory only — not persisted
  to `session.json` (session restore already picks its own active tab).
  A remembered tab that has since closed or left the cluster simply fails
  the lookup and falls back to the cluster's first tab.
- Private tabs participate normally (they live in clusters like any tab).

### Implementation (all in `src/main/main.js`)

1. Extract the slots computation from `selectTabAtIndex` into a shared
   helper (`clusterSlots()`), returning
   `[{ key, group|null, tabIds }, ...]`, so ⌘1–9, ←/→, and ↑/↓ all use one
   definition of cluster order.
2. `cycleTabInCluster(direction)` — find the slot containing `activeTabId`,
   cycle within its `tabIds`. No slots / no groups+pins → `cycleTab(direction)`.
3. `cycleCluster(direction)` — find the active slot index, step with wrap,
   activate `lastActiveByCluster.get(key) ?? tabIds[0]`, uncollapse target
   group, `setActiveTab`.
4. Update `setActiveTab` to record the tab into `lastActiveByCluster` under
   its current cluster key.
5. Four visible menu items in the Tabs menu, next to Next/Previous Tab:
   "Next Tab in Group" / "Previous Tab in Group" (`Alt+CmdOrCtrl+Right/Left`)
   and "Next Group" / "Previous Group" (`Alt+CmdOrCtrl+Down/Up`).

## Part 2: Help menu + `blanc://shortcuts/` page

### Help menu

New last entry in the menu template:

- macOS: `{ role: 'help', submenu: [...] }` — gets Apple's built-in Help
  search field for free.
- Windows/Linux: plain `{ label: 'Help', submenu: [...] }`.
- Single item: **"Keyboard Shortcuts"**, accelerator `CmdOrCtrl+/`, click →
  `openInternalPage('blanc://shortcuts/')` (focuses an existing copy rather
  than opening duplicates, like Downloads/Settings/History).

### The page

- New files `shortcuts.html` + `shortcuts.js`, flat in
  `src/renderer/pages/` (the pages handler only serves flat files), with
  their own CSP `<meta>` tag (same policy as the other pages) and styled
  with `pages.css` tokens — no new stylesheet.
- `'shortcuts'` added to the `KNOWN_PAGES` allowlist in `pages.js` so
  `blanc://shortcuts/` resolves; uses the `bowserPages` bridge
  (`tab-preload.js`) — no new preload.

### Data source: menu introspection + static extras

New guarded IPC `pages:shortcuts:list` in `pages.js` (same
`event.sender.getURL()` origin re-verification as every `pages:*` handler),
backed by a hook from `main.js`:

1. Walk `Menu.getApplicationMenu()`: for every item with an `accelerator`,
   emit `{ category: <top-level menu label>, label, accelerator }`.
2. Skip `visible: false` items (hidden aliases like the silent `⌘=` zoom
   binding) and role-only items whose accelerators the OS provides.
3. Collapse the nine "Tab or Group N" items into one row:
   "Tab or Group 1–9" / `⌘1–9`.
4. Append a static extras list for non-menu keys:
   - Escape — dismiss the island panel / find bar
   - Enter (in command bar) — open address or run command
   - ⌘-click a link — open in background tab
5. Format accelerators to platform display strings in main before sending
   (`⌘⇧T` on macOS, `Ctrl+Shift+T` elsewhere), matching the overlay's
   existing `modKey` convention.

Because the list is read from the live menu, any future shortcut added to
the menu template (including Part 1's four arrow items) appears on the page
automatically — the page cannot drift from the real bindings.

### Slash commands section

The page ends with a static table of the island's slash commands (they are
keyboard-driven too): `/allow-ads`, `/block-ads`, `/clear`, `/close`,
`/close-group`, `/downloads`, `/favorites`, `/find`, `/group <name>`,
`/history`, `/mute`, `/new`, `/pin`, `/private`, `/settings`, `/theme`,
`/ungroup`. This list is hand-maintained in `shortcuts.js`; a one-line
comment in `overlay.js`'s command table reminds editors to update it.

### Layout

Single column, one section per category (File, View, Tabs, Favorites, Help,
Island, Slash Commands), each a simple two-column row list: action label
left, keystroke right in a `<kbd>`-styled capsule. Date-line/header styling
consistent with the other internal pages. No search box — the page is one
screenful and ⌘F works; macOS users also get Help-menu search natively.

## Error handling

- Arrow commands no-op gracefully when there is no active tab, one tab, or
  one cluster — no throws, no beeps.
- `pages:shortcuts:list` from a non-`blanc://` sender is rejected (existing
  guard pattern).
- If the application menu is somehow unset when the page asks, return just
  the extras + slash commands rather than failing.

## Testing

No test suite exists in this repo. Verification is manual via `npm start`
(chrome-level changes need a full relaunch, not ⌘R):

- ←/→ within a multi-tab group wraps; in a single-tab group no-ops; with no
  groups cycles all tabs.
- ↑/↓ visits pinned shelf → each group → ungrouped, wraps, restores each
  cluster's last-active tab, uncollapses collapsed groups.
- ⌘1–9 behavior unchanged.
- Help → Keyboard Shortcuts (and ⌘/) opens the page once and refocuses it
  thereafter; every menu accelerator appears, hidden aliases don't; dark
  mode and private theme render correctly.
