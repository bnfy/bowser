## Fail-closed release controls

- Candidate/Stable mode, platform selection, and Mac architecture selection
  are explicit.
- Notarization failure, workflow dispatch/registration failure, incomplete
  Windows signing, unexpected Windows publisher, missing/stale assets,
  checksum mismatch, and dirty/non-main release inputs stop publication.
- Assets stage in one authenticated draft and are verified before the draft
  is published.
- GitHub Actions use the current published `actions/checkout@v6` and
  `actions/setup-node@v6` majors.

## External gates still open

- GitHub CLI authentication is expired, so repository secret/variable state
  and the corrected Windows/Linux workflows could not be dispatched.
- The corrected workflow files are not on the default branch until the P0
  source is reviewed, committed, and pushed.
- No exact-version notarized RC exists yet, so there are no staged artifact
  hashes or clean-install results to record.
- Native Intel, Windows 11, and Linux x86_64 candidate tests remain absent;
  those targets stay out of the eligible matrix.
- The seven-day no-P0/P1 quiet period starts only after an immutable RC is
  distributed.
